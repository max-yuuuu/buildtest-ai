## 快速开始 (开发模式)

### 1. 克隆项目

```bash
git clone https://github.com/Tencent/WeKnora.git
cd WeKnora
```

### 2. 启动基础设施

```bash
make dev-start
```

这会启动：
- PostgreSQL
- Redis
- MinIO (可选)
- Neo4j (可选)

### 3. 启动后端服务

```bash
make dev-app
```

### 4. 启动前端服务

```bash
make dev-frontend
```

### 5. 访问服务

- 前端: http://localhost:5173
- 后端 API: http://localhost:8080
- Swagger: http://localhost:8080/swagger/index.html

## 项目配置

### 环境变量

复制环境变量示例文件：

```bash
cp .env.example .env
```

主要配置项：

```env
# 服务器配置
SERVER_HOST=0.0.0.0
SERVER_PORT=8080

# 数据库配置
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_USER=postgres
DATABASE_PASSWORD=postgres
DATABASE_NAME=weknora

# Redis 配置
REDIS_HOST=localhost
REDIS_PORT=6379

# LLM 配置 (示例: OpenAI)
LLM_PROVIDER=openai
OPENAI_API_KEY=your-api-key
OPENAI_MODEL=gpt-4

# 存储配置
STORAGE_TYPE=local
```

## 常见开发任务

### 1. 添加新的 API 接口

1. 在 `internal/handler/` 中创建或修改 Handler
2. 在 `internal/router/` 中注册路由
3. 添加 Swagger 注释
4. 运行 `make docs` 生成文档

示例：

```go
// internal/handler/xxx.go

// CreateXXX 创建 XXX
// @Summary 创建 XXX
// @Description 创建新的 XXX
// @Tags XXX
// @Accept json
// @Produce json
// @Param xxx body CreateXXXRequest true "XXX 信息"
// @Success 200 {object} types.Response
// @Router /api/v1/xxx [post]
func (h *XXXHandler) CreateXXX(c *gin.Context) {
    // 实现
}
```

### 2. 添加新的模型提供商

1. 在 `internal/models/provider/` 中创建新的 provider
2. 实现 `ChatProvider` 或 `EmbeddingProvider` 接口
3. 在 `provider.go` 中注册

### 3. 添加新的向量数据库

1. 在 `internal/application/repository/retriever/` 中创建新的 retriever
2. 实现 `Retriever` 接口
3. 在 `registry.go` 中注册

### 4. 添加新的 IM 适配器

1. 在 `internal/im/` 中创建新的平台目录
2. 实现 `Adapter` 接口
3. 在 `service.go` 中注册

## 贡献代码

### 分支策略

- `main`: 主分支，稳定版本
- `develop`: 开发分支
- `feature/xxx`: 功能分支
- `fix/xxx`: 修复分支

### 提交规范

使用 Conventional Commits 规范：

```
<type>(<scope>): <subject>

类型:
- feat: 新功能
- fix: 修复
- docs: 文档
- style: 格式
- refactor: 重构
- test: 测试
- chore: 构建/工具

示例:
feat(agent): add new tool for data analysis
fix(knowledge): fix document parsing error
```

### Pull Request 流程

1. Fork 项目
2. 创建功能分支
3. 提交更改
4. 推送到 Fork
5. 创建 Pull Request

## 常见问题

### 端口被占用

修改 `.env` 中的端口配置，或停止占用端口的进程。

### 数据库连接失败

检查：
- PostgreSQL 容器是否运行
- 数据库配置是否正确
- 网络连接

### 模型 API 调用失败

检查：
- API Key 是否正确
- 网络连接是否正常
- 模型是否可用

### 前端无法连接后端

检查：
- 后端服务是否正常启动
- CORS 配置是否正确
- 前端 API_BASE_URL 配置
