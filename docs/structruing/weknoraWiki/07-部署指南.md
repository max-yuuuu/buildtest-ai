## Docker Compose 部署 (推荐)

### 1. 准备环境

确保已安装：
- Docker 20.10+
- Docker Compose 2.0+

### 2. 克隆项目

```bash
git clone https://github.com/Tencent/WeKnora.git
cd WeKnora
```

### 3. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env 文件，配置你的设置
```

必须配置的项：
```env
# LLM 配置 (至少配置一个)
LLM_PROVIDER=openai
OPENAI_API_KEY=your-api-key

# 管理员账号
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your-password
ADMIN_EMAIL=admin@example.com
```

### 4. 启动服务

#### 最小化部署 (核心服务)

```bash
docker compose up -d
```

这会启动：
- app (后端)
- ui (前端)
- postgres (数据库)
- redis (缓存/队列)

#### 完整部署 (包含所有可选服务)

```bash
docker compose --profile full up -d
```

包含：
- 核心服务
- docreader (文档解析)
- minio (对象存储)
- neo4j (知识图谱)
- jaeger (链路追踪)

#### 按需启用服务

```bash
# 只启用知识图谱
docker compose --profile neo4j up -d

# 只启用 MinIO
docker compose --profile minio up -d

# 同时启用多个
docker compose --profile neo4j --profile minio up -d
```

### 5. 访问服务

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost |
| 后端 API | http://localhost:8080 |
| Swagger 文档 | http://localhost:8080/swagger/index.html |
| MinIO Console | http://localhost:9001 |
| Jaeger | http://localhost:16686 |

### 6. 查看日志

```bash
# 查看所有服务日志
docker compose logs -f

# 查看特定服务日志
docker compose logs -f app
docker compose logs -f ui
```

### 7. 停止服务

```bash
docker compose down

# 停止并删除数据卷 (谨慎使用)
docker compose down -v
```

### 8. 更新服务

```bash
# 拉取最新镜像
docker compose pull

# 重启服务
docker compose up -d
```

## Kubernetes 部署 (Helm)

### 1. 准备环境

- Kubernetes 1.20+
- Helm 3.0+

### 2. 添加 Helm 仓库

```bash
helm repo add weknora https://tencent.github.io/WeKnora
helm repo update
```

### 3. 创建配置文件

创建 `values.yaml`：

```yaml
# 基本配置
ingress:
  enabled: true
  hosts:
    - host: weknora.example.com
      paths:
        - path: /
          pathType: Prefix

# 数据库配置
postgresql:
  enabled: true
  auth:
    username: weknora
    password: your-password
    database: weknora

# Redis 配置
redis:
  enabled: true
  auth:
    password: your-redis-password

# 应用配置
app:
  env:
    LLM_PROVIDER: openai
    OPENAI_API_KEY: your-api-key
    # 更多配置...
```

### 4. 安装 Chart

```bash
helm install weknora weknora/weknora -f values.yaml
```

### 5. 升级部署

```bash
helm upgrade weknora weknora/weknora -f values.yaml
```

### 6. 卸载

```bash
helm uninstall weknora
```

## Lite 版本部署 (零依赖)

Lite 版本是单文件版本，使用 SQLite 作为数据库，无需 Docker。

### 1. 下载二进制文件

从 GitHub Releases 下载对应平台的版本：
- Linux: `weknora-lite-linux-amd64`
- macOS: `weknora-lite-darwin-amd64` / `weknora-lite-darwin-arm64`
- Windows: `weknora-lite-windows-amd64.exe`

### 2. 配置

创建 `.env.lite`：

```env
SERVER_HOST=0.0.0.0
SERVER_PORT=8080

LLM_PROVIDER=openai
OPENAI_API_KEY=your-api-key

ADMIN_USERNAME=admin
ADMIN_PASSWORD=your-password

STORAGE_TYPE=local
STORAGE_LOCAL_PATH=./data
```

### 3. 运行

```bash
# Linux/macOS
chmod +x weknora-lite-*
./weknora-lite-*

# Windows
weknora-lite-windows-amd64.exe
```

### 4. 使用 systemd 管理 (Linux)

创建 `/etc/systemd/system/weknora.service`：

```ini
[Unit]
Description=WeKnora Service
After=network.target

[Service]
Type=simple
User=weknora
WorkingDirectory=/opt/weknora
ExecStart=/opt/weknora/weknora-lite-linux-amd64
Restart=always
EnvironmentFile=/opt/weknora/.env.lite

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable weknora
sudo systemctl start weknora
sudo systemctl status weknora
```

## 生产环境配置

### 1. 安全配置

#### 使用 HTTPS

使用 Nginx 或 Traefik 作为反向代理：

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://localhost:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

#### 加密 API Keys

WeKnora 支持静态加密敏感数据：

```env
# 生成加密密钥 (必须保密)
ENCRYPTION_KEY=your-32-byte-key-here
```

### 2. 数据持久化

#### 数据库备份

PostgreSQL 备份脚本：

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR=/path/to/backups
DATE=$(date +%Y%m%d_%H%M%S)

docker exec weknora-postgres-1 pg_dump -U postgres weknora > $BACKUP_DIR/weknora_$DATE.sql

# 保留最近 7 天的备份
find $BACKUP_DIR -name "weknora_*.sql" -mtime +7 -delete
```

添加到 crontab：

```bash
0 2 * * * /path/to/backup.sh
```

#### 对象存储

建议使用云存储服务：
- 腾讯云 COS
- 阿里云 OSS
- AWS S3

配置示例：

```env
STORAGE_TYPE=s3
S3_ENDPOINT=s3.amazonaws.com
S3_BUCKET=weknora
S3_ACCESS_KEY=your-access-key
S3_SECRET_KEY=your-secret-key
S3_REGION=us-east-1
```

### 3. 高可用配置

#### 数据库主从复制

配置 PostgreSQL 主从复制。

#### Redis 集群

使用 Redis Sentinel 或 Redis Cluster。

#### 应用水平扩展

```yaml
# docker-compose.prod.yml
services:
  app:
    deploy:
      replicas: 3
```

### 4. 监控与日志

#### 日志收集

使用 ELK (Elasticsearch + Logstash + Kibana) 或 Loki + Promtail。

#### 性能监控

使用 Prometheus + Grafana。

### 5. 向量数据库选择

根据规模选择合适的向量数据库：

| 规模 | 推荐 |
|------|------|
| 小型 (< 100k) | pgvector (PostgreSQL) |
| 中型 (100k - 1M) | Milvus / Qdrant |
| 大型 (> 1M) | Milvus / Elasticsearch |

配置示例 (使用 Milvus)：

```env
VECTOR_STORE_TYPE=milvus
MILVUS_HOST=localhost
MILVUS_PORT=19530
```

## 常见问题

### 容器启动失败

检查：
- 端口是否被占用
- 磁盘空间是否足够
- 权限是否正确

### 数据库连接失败

检查：
- PostgreSQL 容器是否正常运行
- 环境变量配置是否正确
- 网络连接是否正常

### 文档解析失败

检查：
- docreader 服务是否正常运行
- 文件格式是否支持
- 文件大小是否超限

### 性能优化建议

- 使用 SSD 存储
- 配置适当的向量索引参数
- 启用缓存
- 使用 CDN 加速静态资源
