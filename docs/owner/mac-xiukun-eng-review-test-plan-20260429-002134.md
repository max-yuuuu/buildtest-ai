## Affected Pages/Routes
- POST /chat/stream (mode=quick) — LLM 流式生成替代模板生成
- POST /chat/stream (mode=agent) — Agent 多步推理（新增）
- GET/PUT /api/v1/knowledge-bases/{kb_id}/model-configs — 知识库模型配置（新增）
- GET/PUT /api/v1/agent-configs — Agent 模型配置（新增）

## Key Interactions to Verify
- Quick Chat: 用户输入问题 → LLM 流式回答 → citation 卡片渲染
- Agent Chat: 用户输入问题 → 多步推理 → 工具调用展示 → 最终回答
- 模式切换: Quick ↔ Agent 切换，历史消息保留
- 模型配置: 按用途选择不同 Provider 的模型

## Edge Cases
- 空查询输入 → 错误提示
- 检索无结果 → "未找到相关信息"降级回答
- 单 KB 检索失败 → 不阻断其他 KB
- LLM 超时 30s → yield 错误消息，不重试
- LLM API key 无效 → 明确错误提示
- Agent 达到 3 次迭代上限 → 强制 final_answer
- 对话历史超过 20 条 → 截断最早的
- 上下文窗口超限 → 滑动窗口裁剪
- 无 LLM provider 配置 → 提示用户配置
- 跨 Provider 选模型（OpenAI embedding + Anthropic LLM）

## Critical Paths
- [→E2E] 上传文档 → 建知识库 → Quick Chat 问问题 → 得到 LLM 回答（端到端）
- [→E2E] Agent Chat 多步推理 → 调用 knowledge_search → 给出回答
- [→E2E] 知识库设置页 → 配置不同 Provider 的 embedding 和 LLM 模型
- [→EVAL] Prompt 模板变更后回答质量不退化

## Regression Risks
- test_quick_chat_workflow.py — 测试旧 QuickChatWorkflow 类，LangGraph 替换后类不存在
- test_chat_stream_contract.py — 测试旧 stream() 事件顺序，重构后事件流变化
- test_chat_mode_routing.py — mode 分发逻辑可能变化
